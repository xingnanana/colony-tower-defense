# 🏘 村庄模拟塔防

部落冲突风格的俯视网格塔防游戏，**白天生产建设，夜晚抵御怪物**。纯 HTML/CSS/JS 单文件（约 2000+ 行），浏览器直接打开即玩。

## 🎮 快速开始

浏览器打开 `game.html` 即可，无需安装任何依赖。

## 🕹 操作

| 操作 | 方式 |
|------|------|
| 放置建筑 | 右侧面板选择 → 左键地图空白处 |
| 建筑交互 | 左键已放置的建筑 |
| 移动建筑 | 右键菜单 → 移动 → 左键确认位置 |
| 移动视角 | 左键拖拽空地 |
| 缩放 | 滚轮（以光标为中心，0.3x ~ 3x） |
| 取消/关闭 | 右键 |
| 铺设地板 | 选中地板 → 单击放置 5×5 地板（1 木/次） |
| 框选砍树 | 🪓 砍伐模式 → 左键拖拽框选树木 |
| 取消砍树标记 | ❌ 按钮 → 左键拖拽框选取消 |

当前游戏界面使用三栏运营布局：左侧开发控制、中央地图、右侧建筑目录。游戏内建筑使用几何徽标和 Canvas 图形，不依赖 emoji 字体；本 README 中的符号仅用于文档说明。

## 🏗 建筑系统

### 解锁等级

升级大本营逐步解锁新建筑，建筑等级上限 = 大本营等级：

| 大本营等级 | 升级消耗 | 解锁建筑 |
|-----------|---------|---------|
| Lv.1 | — | 农场、产房、箭塔、地板 |
| Lv.2 | 20 木 | 住房、兵营、练兵房、木材仓库、食物仓库 |
| Lv.3 | 50 木 | 采石场、石材仓库、城墙 |
| Lv.4 | 200 木 + 50 石 | 铁矿、炼铁机、铁矿仓库、铁锭仓库 |
| Lv.5 | 500 木 + 200 石 | 自动箭塔 |

### 建筑交互菜单

点击已放置建筑弹出菜单：
- **升级** — 消耗资源提升建筑等级。除生命值外，不同类型会获得专属成长：生产设施每级 +20% 生产速度和 +2 暂存；仓库每级 +25 容量；人工箭塔每级 +25% 伤害；自动箭塔每级 +20 射程；灯每级 +1 格视野；城墙每级额外承受 15% 减伤（上限 75%）。住房 +2 人口，大本营 +4 人口 + 解锁新建筑
- **增减工人** — 分配/移除生产工人（不可超过建筑上限）
- **移动** — 半透明虚影拖到新位置（大本营不可移动）
- **拆除** — 删除建筑（大本营不可拆除）
- 产房独有：**招募村民**（5 食物 / 人，10 秒）。
- 练兵房独有：**招募守卫**（20 食物 / 人，15 秒）。
- 住房和大本营只提供村民床位；兵营提供守卫床位。大本营初始可容纳 2 名守卫，开局已有 1 名守卫。

### 蓝图建造流程

放置建筑 → 创建蓝图（半透明，HP=1，不显示常规 UI）→ **工程师**从仓库取材料（每次 ≤5 个）→ 运到蓝图交付 → 材料交齐 → 建造倒计时 → **建筑完成**（HP 恢复，住房人口生效）。

建造时间按解锁等级递增：Lv.1 = 2s，Lv.2 = 5s，Lv.3 = 8s，Lv.4 = 11s，Lv.5 = 14s。

## 👥 居民

### 行为优先级（从高到低）

1. **饥饿** → 去最近食物仓库 / 大本营取食
2. **困倦** → 回家（住房 / 大本营）睡觉（到家隐藏）
3. **有工作** → 去工作场所 → 生产 → 搬运产物到对应仓库 → 循环
4. **空闲** → 有标记树木优先砍树 → 否则只在已完成建筑的稳定视野内白天巡逻，夜晚回家

生产效率 = 分配人数 / 所需工人（上限 100%）。

生产建筑内部会暂存最多 5 个产物；达到 5 个后由一名工人整批搬运。护林员会连续砍树，木材携带量达到 5 个后再返回仓库。

护林员小屋会在辐射范围内的空格自动补种树苗，无需工人在场；范围被树木、树苗或其他实体占满时停止补种。树苗使用错开的成长时间成熟，成熟树由分配到小屋的工人采伐。

### 初始配置

初始拥有 `START_POP` 名村民，全部住大本营，其中前两人为工程师；另有 1 名守卫，同样驻扎在大本营。村民与守卫上限分开计算。

## 迷雾

世界开局被完整迷雾覆盖，只有“可见”和“不可见”两种状态，不保留最后一次所见的地图快照。大本营驱散半径 8 格，灯驱散半径 5 格；其他已完成建筑仅驱散自身外接圆。居民和守卫会提供动态视野，半径分别为 2 格和 3 格。

夜晚时，大本营和已完成的灯会在夜幕遮罩中局部恢复接近白天的原始色彩，中心到边缘缓慢过渡。视觉光照半径与各自迷雾驱散半径一致，不改变实际视野或迷雾规则。

建筑、地板的全部占地必须处于可见区域才能放置。敌人在入夜时从玩家视野边界外、至少深入 2 格的迷雾中预定生成；迷雾中的敌人不会绘制，也不会成为箭塔目标，敌人仍按正常 AI 行动。

带有作用范围的建筑在放置蓝图、移动预览和选中已建成建筑时，都会显示圆形覆盖范围。灯显示点亮范围，护林员显示育林范围，矿场显示资源搜寻范围，箭塔显示攻击范围。

### 移动方式

居民采用连续坐标移动，直线受建筑阻挡时会生成绕行点；建筑布局变化会使旧路线失效，移动中每 0.25 秒验证路线，卡住时自动重算。建筑、矿石和居民使用空间哈希缩小查询范围。地板可提供 1.5x 移速加成，当前尚未启用完整网格寻路或 NavMesh。

## ⚔️ 守卫系统

守卫是独立兵种，不同于普通居民。白天睡觉，夜晚战斗。

### 属性

| 属性 | 数值 |
|------|------|
| HP | 80 |
| 攻击力 | 12 |
| 攻击 CD | 0.6s |
| 视野 | 450px |
| 招募消耗 | 20 食物 / 人（15 秒） |

### AI 行为

1. **GUARD_FIND_TOWER** → 前往玩家分配的箭塔
2. **GUARD_MANNING** → 站在箭塔上，自动攻击范围内敌人
3. **GUARD_FIGHTING** → 主动搜寻 450px 内敌人交战，优先保护被攻击建筑
4. **GUARD_RETURNING** → 受伤回大本营
5. **GUARD_HEALING** → 治疗 2 游戏小时后满血重返战场

巡逻范围限制在建筑群外围，白天自动回家睡觉（隐藏+回满血）。

> **箭塔需要分配 1 名守卫**才能开火！选中箭塔后，在建筑菜单中增加或减少守卫。未分配的守卫负责机动防守；Lv.5 解锁的自动箭塔不需要守卫。

## 🌲 地图资源与砍伐

### 自然资源

初始地图资源以大本营为圆心生成：树木在半径 4 格外，石头在 10 格外，铁矿在 20 格外。初始可见范围内至少保证 5 棵树。树木按 1 至 10 棵的簇生成，目标平均 4；石头按 1 至 6 块的簇生成，目标平均 1.5；铁矿通常单个生成，但会低概率形成 2 至 4 块的小簇。

树木可以通过砍伐指令直接采集；石矿和铁矿不能由居民直接开采，必须在资源点附近建造采石场或铁矿，再安排工人生产。

石矿和铁矿会阻挡单位移动并被寻路绕开；树木和树苗不设置移动碰撞体。

### 砍伐系统

- 点击 🪓 按钮进入砍伐模式 → 左键拖拽框选树木标记
- 空闲非工程师村民自动前往砍树（3 秒/棵）
- 砍倒后获得木材，村民搬运至木材仓库
- ❌ 按钮取消砍伐标记

## 🗄 仓库与资源

五种资源：🍗 食物、🪵 木材、🪨 石材、⚒ 铁矿、🔩 铁锭。

- 每种资源只能放入对应仓库（+ 大本营兜底）
- 专用仓库总容量 50，大本营总容量 200；仓满后生产和搬运会等待
- 顶部全局资源是具体仓库库存的汇总，吃饭、建造和炼铁都会扣除实际库存
- 仓库进度条颜色：绿 < 50% → 黄 50-80% → 橙 > 80%
- 顶部全局资源栏汇总所有仓库存储量

## 🌙 昼夜系统

一天共 256 秒（4 分 16 秒）：

| 时段 | 时长 | 说明 |
|------|------|------|
| ☀️ 白天 | 162s（6:00 ~ 22:00） | 居民生产建设，守卫睡觉 |
| 🌅 黄昏 | 8s | 居民回家，守卫起床，怪物开始刷新 |
| 🌙 夜晚 | 78s（22:00 ~ 6:00） | 怪物进攻，守卫和箭塔防御 |
| 🌄 黎明 | 8s | 怪物撤退，守卫回家睡觉，居民起床 |

右上角圆形 24 小时时钟指示当前时间。

## ⚔️ 怪物与战斗

- 夜晚开始时按当前视野形状建立迷雾前线候选点。总数默认 = 3 + 天数 × 2，并会在多波的生成时间窗口内随机错峰出现；预定点被照亮时对应敌人等待，不会改点或直接出现在清晰区。
- **普通敌人**：40 生命、50 移速、8 伤害，所有夜晚都会出现。
- **快速敌人**：第 3 天起按权重混入，24 生命、88 移速、5 伤害；用更快的速度考验防线覆盖范围。
- **破坏者**：第 6 天起按权重混入，110 生命、30 移速、16 伤害；优先攻击路线上的城墙和防御建筑。
- 优先攻击守卫；无人阻挡时直冲大本营。建筑会按其是否位于敌人至大本营的直线路径上判定，优先攻击最先挡路的建筑；多个等价目标时选择最近者，因此城墙和前线建筑能稳定承担阻挡作用
- 箭塔（需守卫操纵）和自动箭塔会向范围内可见敌人发射实体箭矢；箭矢抵达目标后才造成伤害
- 箭塔守卫会前往箭塔下方的固定驻守点；仓库和蓝图交互恢复以建筑中心为目标
- 天亮怪物撤退
- 无家可归的居民夜晚会被怪物攻击杀死
- 大本营被摧毁 → 游戏结束

## ✏️ 配置编辑器

调试面板 → ✏️ 编辑配置，弹出数值编辑器：

- 编辑建筑属性采用按建筑类型定制的字段：生产设施显示工人、周期和产物，仓库显示容量，箭塔显示射程、伤害和攻击间隔；灯显示“迷雾驱散范围”，护林员与矿场显示各自覆盖范围，产房与练兵房显示招募时间和消耗。
- 建筑页可选择等级并配置该等级的覆盖参数；每种建筑均可设置等级上限。大本营的等级 1 页面还包含迷雾驱散范围、开局资源和食物/木材/石材/铁矿/铁锭的独立仓储上限。
- 仓库、箭塔、自动箭塔、灯、城墙和各生产建筑的升级成长均在对应建筑配置页编辑，不再位于“全局数值”。
- 游戏内资源消耗统一显示为数量加资源图标，建筑目录、升级与招募菜单不再使用“木/石/食物”等资源文字缩写。
- 通过“大本营解锁”分类配置每种建筑首次可建造所需的大本营等级
- 通过“敌人生成与波次”分类配置每夜数量成长、波数、波次节奏、随机度和迷雾生成深度；通过“敌人类型”分类配置三种敌人的属性、首次出现天数和生成权重；修改后下一次夜晚生效
- 通过“初始地图资源”分类配置自然资源总量、簇数量、最小生成半径、可见树木下限和簇分布；重新开始游戏后生效
- “初始人口”分类只管理开局人口与人口上限；开局资源已迁移至大本营的建筑配置。
- 修改即时生效（已有建筑 HP 同步更新、面板文字刷新）

- 使用 VS Code 内嵌预览时，运行“Browser: Open Integrated Browser”并输入上述地址；不要使用 Live Preview 的 3000 静态服务，否则无法保存项目数值配置。
- `.vscode/launch.json` 提供“调试村庄模拟（内嵌浏览器）”：在“运行和调试”面板选择它并按 `F5`，VS Code 会先启动 3010 服务，再在内嵌浏览器中打开游戏并支持 JavaScript 调试。
- 若 3010 服务已经运行，任务会复用它而不会因端口占用失败。
- “应用”只更新当前运行中的游戏；“保存到项目”会写入 `data/balance.json`，刷新后自动加载。
- 直接打开 `game.html` 仍可游玩和临时调参，但浏览器安全限制下不能写入项目文件。
- 从不支持保存的页面点击“保存到项目”时，编辑器会直接提供“打开可保存的开发版本”链接，无需手动输入地址。
- 全局字段会标明即时生效或重开生效；项目配置统一保存在 `data/balance.json`。

### 游戏设置与快捷键

- 顶部 HUD 的“设置”可修改快捷键，修改后自动保存到当前浏览器。
- 默认快捷键：`空格` 暂停/继续，`1` 正常速度、`2` 两倍速度、`3` 四倍速度，`Esc` 取消当前操作，`C` 伐木标记，`X` 取消伐木标记，`F1` 打开游戏设置，`F2` 打开数值编辑器。
- 同一按键不能绑定到两个操作；重新绑定时，原操作会恢复默认快捷键。设置面板可一键恢复默认。

# 殖民模拟塔防游戏 — 交接文档

## 项目概况

- **项目名**: 殖民模拟塔防游戏
- **文件**: 单文件 `game.html`（当前约 2600+ 行）
- **技术栈**: 纯 HTML5 Canvas + Vanilla JS（无框架）
- **仓库**: `github.com/xingnanana/colony-tower-defense`
- **设计参考**: Kingdoms and Castles

## 当前游戏状态（2026-07-11）

### 地图
- **150×150 格**，40px/格，6000×6000 像素世界
- 相机支持拖拽平移 + 滚轮缩放（0.3x-3x）
- 初始：大本营在 (74, 74)，仅此一个建筑

### 人口系统
- **工人**: 大本营招募（5食物/10秒）
- **守卫**: 大本营招募（20食物/15秒），独立人口类型
- **初始**: 4人，前2人默认工程师
- 人口上限由住房+大本营决定

### 守卫系统
- HP=80，攻击=12/0.6s，搜索范围 450px
- 不吃饭不睡觉，白天隐藏睡觉，夜晚出来
- AI: 被分配箭塔的守卫前往操控；未分配守卫机动战斗 → 受伤回大本营回血
- 巡逻范围：所有建筑外接圆 + 100px
- 敌人攻击建筑时，巡逻守卫优先冲过去打

### 地板系统（新增）
- 5×5格，1木/块，单击放置
- 独立 `G.floorMask` 网格（不在 buildings 中）
- 1.5倍移速加成
- 可放在任何位置（空地/建筑上/已有地板上）
- 底层渲染（地面层级）

### 当前界面与美术
- 游戏界面采用左侧开发控制栏（168px）、中央地图、右侧建筑目录（188px）的三栏布局
- 建筑目录按基础、生产、仓储、防御分区，生产/仓储/防御使用中文单字徽标；地板、住房和大本营不显示标识
- 顶部资源栏使用 CSS 颜色与形状标记资源、人口、守卫和昼夜
- Canvas 建筑使用几何结构绘制，资源点使用树木和矿石图形绘制
- 施工班组和林务操作集中在地图右下角，状态提示使用中文文字与颜色

### 生产与仓储
- 生产进度按建筑统一结算，不再由每名工人重复推进
- 效率按当前在岗工人计算，搬运中的工人不计入生产效率
- 生产建筑产物在建筑内部暂存，容量为 5；满 5 后由一名工人整批搬运，搬运者不计入在岗效率
- 护林员连续砍树并批量携带木材，附近没有下一棵树时允许携带不足 5 个收尾搬运
- 普通生产批次记录来源建筑；搬运未完成时不会因仓库竞争、昼夜切换或居民移除而直接丢失，未送达部分继续搬运或退回建筑缓存
- 仓库容量真实生效；全局资源栏是各仓库库存汇总
- 居民吃饭、工程师取料、建筑消耗统一扣除具体仓库库存
- 采石场/铁矿必须建在对应矿点 3 格半径内，资源点可持续生产
- 炼铁机每生产 1 铁锭消耗 1 铁矿
- 只有树木允许居民直接砍伐；石矿和铁矿不能被手工采集，必须由对应矿场生产

### 护林员与农场连携
- 护林员小屋工人在范围内种树、砍树并搬运木材
- 树苗 10 秒成熟，自动归属对应护林员小屋
- 农场四面贴邻每面 +15% 产量，并显示连携线

### 砍伐系统
- 🪓砍伐 / ❌取消框选按钮（右下角 engineer bar 左侧）
- 框选标记树木 → 空闲村民自动砍（3秒/棵）
- 砍完自动搬运到仓库 → 找下一棵标记树
- 多人不重复砍同一棵树
- `marked` 标记控制，取消后村民立即放弃

### 地图自然资源
- `G.resourceNodes[]`：树(~150)、石矿簇(~25×2-3)、铁矿簇(~15×2-3)
- 石矿和铁矿是移动障碍并参与绕行检测；树木和树苗不阻挡移动
- 资源在 `isCellBlocked` 中阻止建筑放置
- 树: 1×1，石/铁: 各占 1 格

### 寻路系统（已简化）
- 简单直线移动 + 碰撞推开
- `resolveCollisions`: 跳过 home/workplace/buildTarget 建筑
- 之前尝试过流场/Dijkstra/道路优先，均因 bugs 回退

## 关键全局状态 (`G`)

```js
G = {
  // 游戏循环
  tick, dt, day, dayTime, phase('day'|'night'|'dusk'|'dawn'),

  // 资源
  resources: {food, wood, stone, iron, ingot},

  // 实体
  buildings[], residents[], enemies[], particles[],
  townHall,  // 大本营引用
  resourceNodes[],  // [{type:'tree'|'stone'|'iron', col, row, x, y, alive, marked}]

  // 地板
  floorMask: Uint8Array[150*150],

  // 砍伐
  chopMode, unchopMode, chopStartX/Y/EndX/Y,

  // 相机
  cam: {x, y, zoom},

  // 交互
  selectedBldType, selectedBuilding, placingMode, movingBuilding,
  hoveredCell, mouseX, mouseY, dragging,

  // 人口
  maxPop,

  // 敌人
  enemySpawnQueue[], enemySpawnTimer,
}
```

## 关键函数

| 函数 | 作用 |
|------|------|
| `initGame()` | 初始化：放建筑、创居民、生成资源、initFloorMask |
| `restartGame()` | 重置所有状态后调 initGame |
| `update(dt)` | 主循环：updateDayNight→updateBuildings→updateResidents→enemies/towers→particles |
| `render()` | 主渲染：地面→地板→网格→建筑→资源→敌人→居民→粒子→放置悬停→时钟 |
| `resolveCollisions(r, nx, ny)` | 碰撞解析，跳过 home/workplace/buildTarget/道路 |
| `flowMove(r, tx, ty, speed, dt)` | 直线移动（带地板移速加成） |
| `moveViaFlow(r, tx, ty, speed, dt)` | 封装 flowMove + 赋值 |
| `addBuilding(type, col, row, skipBlueprint)` | 放建筑 |
| `demolishBuilding(b)` | 拆建筑 |
| `generateResources()` | 生成树/石/铁矿 |
| `paintFloorCell(col, row)` | 放 5×5 地板 |
| `toggleChopMode()` / `toggleUnchopMode()` | 砍伐模式切换 |
| `isCellBlocked(col, row, excludeBld)` | 格子检测（含资源节点） |
| `canPlaceBuilding(type, col, row, excludeBld)` | 放置检测 |
| `depositToStorage` / `withdrawFromStorage` | 统一仓库存取与全局总量同步 |
| `findResourceNode(b, type)` | 查询矿场范围内可开采资源 |
| `refreshFarmAdjacency()` | 布局变化时重算农场连携缓存 |

## 居民状态机

```
SLEEPING → (天亮) → IDLE/PATROL
IDLE/PATROL:
  - 工程师有蓝图 → GATHERING → BUILDING → CONSTRUCTING
  - 有工作 → GOING_TO_WORK → WORKING → HAULING → 循环
  - 非工程师有标记树 → GOING_TO_CHOP → CHOPPING(3s) → HAULING
  - 无 → 巡逻
GOING_TO_EAT → EATING（优先）
GOING_HOME → SLEEPING（优先）
```

## 守卫状态机

```
白天: GUARD_SLEEPING (hidden)
黄昏: GUARD_SLEEPING → GUARD_FIND_TOWER
  GUARD_FIND_TOWER:
    已分配箭塔 → GUARD_MANNING（塔启动攻击）
    未分配 → GUARD_FIGHTING（寻敌肉搏）
  GUARD_FIGHTING:
    HP ≤ 0 → GUARD_RETURNING → GUARD_HEALING(2h) → GUARD_FIGHTING
黎明: 任意状态 → GUARD_SLEEPING（回满血）
```

## 代码规范

- 文件: 单文件 HTML
- 风格: 紧凑无注释，函数内联
- 命名: `bl` (building), `r` (resident), `def` (BLD_DEFS entry)
- 建筑定义: `BLD_DEFS` 对象，key 为类型名
- 碰撞: `RESIDENT_RADIUS = 7`
- 文档同步: 遵循 `文档维护规范.md`；代码、交接、README、术语、技术实现、TODO 和测试清单须在同一批改动中同步
- 界面图标: 游戏内使用 CSS/Canvas 几何徽标，emoji 仅允许出现在非界面说明文档中
- 大本营视觉: 不显示底部仓储容量条和住户条；建筑血条仅在当前 HP 低于最大 HP 时显示

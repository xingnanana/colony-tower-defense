## 生产公式

生产建筑产物缓存：`pendingOutput`，默认容量为 `CFG.PRODUCTION_BATCH_SIZE = 5`；`outputHauler` 表示当前负责整批搬运的居民。

生产建筑原料缓存：`productionInputs`；`inputHaulers` 表示当前负责从仓库向建筑并行备料的工人集合，每人通过 `productionInputReservedAmount` 最多预留 5 单位。带配方建筑的整轮需求为 `inputs × productionBufferCapacity()`，必须整轮备齐后才能生产。

```
生产效率 = min(1, activeWorkers / maxWorkers)
实际生产时间 = baseTime / 效率
```

| 参数 | 说明 |
|------|------|
| `baseTime` | 100% 效率下的生产时间（秒） |
| `maxWorkers` | 最大工人数（也等于 100% 效率所需人数） |
| `assignedWorkers` | 当前分配到建筑的工人数 |
| `activeWorkers` | 当前已到岗且处于 WORKING 状态的工人数 |
| `productionProgress` | 0~1，达到 1 时产出 1 个资源 |

生产进度按建筑统一推进一次；工人只负责到岗和搬运，不会各自重复增加建筑进度。农场连携与饥饿减益会作为效率乘数参与结算。

护林员小屋例外：树苗由建筑在辐射范围空格内自动生成，工人只处理成熟树的砍伐和搬运；范围无空格时停止补种。

## 居民 AI 优先级

```
1. 固定用餐时刻到达 → GOING_TO_EAT → EATING
2. 黄昏或夜晚 → GOING_HOME → SLEEPING
3. 有工作 → GOING_TO_WORK → WORKING → HAULING → 循环
4. 工程师(有蓝图) → GATHERING → BUILDING → CONSTRUCTING
5. 工程师(无蓝图) → 巡逻
6. 无业游民 → 巡逻
```

固定用餐时刻到达后，`CHOPPING`、`HAULING`、`GATHERING`、`BUILDING`、`CONSTRUCTING` 属于可收尾任务：居民完成当前树木、搬运或当前建筑后再去吃饭。`WORKING`、`GOING_TO_WORK`、`GOING_TO_CHOP` 等持续岗位或未开始任务会立即前往吃饭。

# 游戏设置与快捷键

- **游戏设置**：保存当前浏览器的操作偏好，不写入项目数值配置。
- **游戏存档**：`village-game-saves-v1` 中的版本化运行时快照，固定最多 10 个槽位，与开发者数值配置相互独立。
- **存档槽**：设置面板中的 `save-slot`，支持保存/覆盖、读取和删除；空槽不可读取或删除。
- **引用恢复**：实体先按数组编号创建，再恢复 `workplace`、`home`、`inputHaulers`、驻塔和目标等对象关系。
- **快捷键录制**：在游戏设置中点击一项操作后，按下新的单个按键完成绑定。
- **速度快捷键**：默认使用主键盘 `1`、`2`、`3` 对应正常、2 倍、4 倍速度；空格负责暂停/继续。

# 代码模块

- **配置层**：`js/config.js`，维护默认 `CFG`、`BLD_DEFS`、设置与全局状态。
- **世界与寻路层**：`js/world.js`、`js/navigation.js`、`js/tasks.js`，维护迷雾、资源、空间查询、路径和任务公共函数。
- **模拟层**：`js/simulation.js`、`js/combat.js`、`js/residents.js`、`js/buildings.js`、`js/game-controls.js`，维护游戏规则更新。
- **表现与交互层**：`js/render.js`、`js/ui.js`、`js/input.js`，维护 Canvas 绘制、菜单和玩家输入。
- **启动入口**：`js/main.js`，负责初始化和主循环；`game.html` 只负责页面结构及脚本顺序。

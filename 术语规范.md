# 术语规范

## 界面术语

| 术语 | 英文（代码） | 说明 |
|------|-------------|------|
| 网页 | — | 整个浏览器页面 |
| 地图 | canvas / `#game` | 游戏画布，1200×800 像素 |
| 调试栏 | `#debug-panel` | 地图外左侧，168px 宽 |
| 建筑栏 | `#building-panel` | 地图外右侧，188px 宽 |
| 总资源栏 | `#topbar` | 地图内中上顶端 |
| 工程师栏 | `#eng-bar` | 地图内右下角 |
| 菜单/弹窗 | `#context-menu` | 左键点击建筑弹出的操作菜单 |
| 配置编辑器 | `#config-overlay` | 全屏弹窗，编辑建筑/资源配置 |
| 游戏结束弹窗 | `#gameover-overlay` | 大本营被摧毁后显示 |
| 时钟 | `drawClock()` | Canvas 绘制，地图右上角 |

界面图标使用 CSS 徽标、Canvas 几何图形和颜色编码；游戏界面不使用 emoji 作为功能图标。

## 建筑术语

| 术语 | 代码 key | 分类 |
|------|---------|------|
| 建筑 | Building | 所有可放置结构的总称 |
| 大本营 | `town_hall` | facility（核心） |
| 生产建筑 | farm/lumber/quarry/iron_mine/smelter | production |
| 仓库/储具 | food_storage/wood_storage/stone_storage/iron_storage/ingot_storage | storage |
| 塔防建筑 | arrow_tower | defense |
| 防御建筑 | wall | defense |
| 设施建筑 | house | facility |
| 地板 | floor | 5×5格地面覆盖，独立 G.floorMask 网格，1.5x移速 |
| 占地面积 | `sz` | 建筑占据的网格格子数，如 `[2,2]` |
| 碰撞面积 | `collisionRadius()` | 圆形碰撞体半径，1×1=14px，2×2=34px |
| 蓝图 | `blueprint` | 建造中的建筑，50% 透明，HP=1 |

## 居民术语

| 术语 | 英文（代码） | 说明 |
|------|-------------|------|
| 居民/村民 | Resident | 所有可控角色 |
| 工人 | (workplace != null) | 被分配到生产建筑的居民 |
| 无业游民 | IDLE / PATROL + !isEngineer | 无工作且非工程师的居民 |
| 工程师 | `isEngineer = true` | 专职建造的居民（独立职业） |
| 活跃工程师 | GATHERING / BUILDING / CONSTRUCTING | 正在取材料/交付/施工 |
| 空闲工程师 | isEngineer + 巡逻中 | 无蓝图时的工程师 |
| 施工业 | CONSTRUCTING | 材料交齐后站在建筑旁等待施工完成 |
| 碰撞体 | `RESIDENT_RADIUS` | 居民脚底碰撞圆半径 = 7px |
| 守卫 | `isGuard = true` | 独立人口类型，大本营招募，不吃饭，夜晚战斗 |
| 箭塔守卫岗位 | `assignedTower` | 选中箭塔后分配 1 名守卫；守卫夜晚前往操控，白天保留岗位 |
| 伐木（砍树）| GOING_TO_CHOP / CHOPPING | 框选标记树后空闲村民自动砍伐(3s/棵) |

## 资源术语

| 术语 | 代码 key | 全局变量 | 对应仓库 |
|------|---------|---------|---------|
| 食物 | `food` | `G.resources.food` | food_storage / town_hall |
| 木材 | `wood` | `G.resources.wood` | wood_storage / town_hall |
| 石材 | `stone` | `G.resources.stone` | stone_storage / town_hall |
| 铁矿 | `iron` | `G.resources.iron` | iron_storage / town_hall |
| 铁锭 | `ingot` | `G.resources.ingot` | ingot_storage / town_hall |

`building.stored` 是具体库存，`G.resources` 是所有有效仓库库存的汇总缓存。专用仓库总容量为 50，大本营总容量为 200。

## 生产公式

生产建筑产物缓存：`pendingOutput`，默认容量为 `CFG.PRODUCTION_BATCH_SIZE = 5`；`outputHauler` 表示当前负责整批搬运的居民。

```
生产效率 = min(1, activeWorkers / maxWorkers)
实际生产时间 = baseTime / 效率
```

| 参数 | 说明 |
|------|------|
| `baseTime` | 100% 效率下的生产时间（秒） |
| `maxWorkers` | 最大工人数（也等于 100% 效率所需人数） |
| `assignedWorkers` | 当前分配到建筑的工人数 |
| `activeWorkers` | 当前已到岗且处于 WORKING 状态的工人数 |
| `productionProgress` | 0~1，达到 1 时产出 1 个资源 |

生产进度按建筑统一推进一次；工人只负责到岗和搬运，不会各自重复增加建筑进度。农场连携与饥饿减益会作为效率乘数参与结算。

## 居民 AI 优先级

```
1. 饥饿 (hunger >= HUNGER_THRESHOLD) → GOING_TO_EAT → EATING
2. 困倦 (tired >= TIRED_THRESHOLD) → GOING_HOME → SLEEPING
3. 有工作 → GOING_TO_WORK → WORKING → HAULING → 循环
4. 工程师(有蓝图) → GATHERING → BUILDING → CONSTRUCTING
5. 工程师(无蓝图) → 巡逻
6. 无业游民 → 巡逻
```
